# Beta Findings Log

Live log of all incidents, confusion patterns, and observations during testnet beta.

---

## How to Use

Each entry has a category prefix:

- `BUG-nn` — Protocol or code bug
- `UX-nn` — User confusion or bad experience
- `ALERT-nn` — Bot alert incorrect, late, or missing
- `INFRA-nn` — Infrastructure issue (RPC, deploy, database)
- `DOCS-nn` — Documentation gap or misleading text

Each entry template:

```
### [CATEGORY]-[nn]: Short title

- **Date:** YYYY-MM-DD
- **Severity:** Critical / High / Medium / Low
- **Screen/Action:** Where it happened
- **Expected:** What user/system expected
- **Actual:** What happened
- **Root cause:** Why
- **Fix:** What was done (or "pending")
- **Preventive:** How to prevent recurrence
- **Status:** Open / Fixed / Won't Fix
```
